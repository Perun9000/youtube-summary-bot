# i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Мультиязычность бота на 7 языков (ru/en/es/fa/pt/ar/id): автодетект по `language_code` Telegram, команда `/language`, все пользовательские строки из JSON-локалей, саммари и Telegra.ph-конспект на языке пользователя, кэш по (video_id, lang).

**Architecture:** Новый модуль `app/i18n.py` (`t(key, lang, **fmt)` + загрузка `app/locales/{lang}.json` при импорте; fallback lang→en→сам ключ) и `UserLangStore` (таблица `user_langs`; выбор: manual > auto-detect > en). Язык едет с задачей (`SummaryJob.lang` + колонка `jobs.lang`); pipeline/delivery/status строят тексты через `t(key, job.lang)`. Summarizer получает `output_lang` и добавляет языковую директиву к system-промпту; канонизация тегов остаётся только для ru. Кэш саммари: ключ `video_id` для ru (legacy-совместимость) и `video_id:lang` для остальных.

**Tech Stack:** Python 3.11, stdlib json, существующий app/db.py, pytest.

## Global Constraints

- `SUPPORTED_LANGS = ("ru", "en", "es", "fa", "pt", "ar", "id")`; `DEFAULT_LANG = "en"` (fallback для неизвестных language_code; у русскоязычных language_code="ru" → автодетект даст ru).
- Локали — JSON-файлы `app/locales/{lang}.json` в репозитории (плоский словарь key→text). В БД — только выбор пользователя (`user_langs`). Никаких переводов в SQLite.
- РУССКИЕ тексты — канон: они берутся ИЗ ТЕКУЩЕГО КОДА ДОСЛОВНО (таблица ключей ниже указывает место). en переводится с ru; es/fa/pt/ar/id переводятся с en.
- Правила перевода (обязательны, проверяются structural-тестом и ревью): плейсхолдеры `{name}` сохраняются буквально и все; /команды (`/subscribe`, `/limits`, …) не переводятся; HTML-теги (`<b>`, `<i>`, `<code>`, `<a href=…>`) сохраняются; эмодзи сохраняются; числа — западные цифры во всех языках (включая fa/ar); тон нейтрально-дружелюбный, обращение: ru «ты», en «you», es «tú», pt «você», fa/ar/id — нейтрально-вежливое.
- Owner-поверхности НЕ переводятся: owner-команды (/stats, /users, /scan_now, /refund, /prompt_*, /llm_*, /cache_drop, /queue, /models, /model, owner-ветка /help), утренний дайджест, уведомления мониторинга, pinned-дайджест — остаются русскими. Технические тексты ошибок для owner/allowlist — русские как сейчас.
- Язык scheduled-задач мониторинга — всегда "ru" (owner-контекст).
- Резолюция языка пользователя: manual-выбор из user_langs → сохранённый auto → `normalize_language_code(from_user.language_code)` (записывается как auto) → "en". Allowlist-пользователи проходят ту же резолюцию (у них language_code=ru → всё как раньше).
- Кэш: записи ru живут под старым ключом `video_id` (обратная совместимость с существующим кэшем), остальные — `{video_id}:{lang}`; `/cache_drop` и колбэк-очистка удаляют ВСЕ языки ролика.
- Тесты: сейчас 100/100; после плана ~112 (точное число зафиксирует последняя задача). Вывод pytest чистый.
- Коммиты английские, в конце `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.

- [ ] **Step 3: app/i18n.py**

```python
"""Локализация пользовательских строк.

Переводы живут в app/locales/{lang}.json (плоский key→text) и грузятся при
импорте — это часть кода, ревьюится в git и деплоится атомарно (осознанное
решение против переводов в БД). В БД — только выбор языка пользователя
(user_langs). Русский — канонический источник; en — база для остальных.
"""
from __future__ import annotations

import json
import logging
from pathlib import Path

logger = logging.getLogger(__name__)

SUPPORTED_LANGS: tuple[str, ...] = ("ru", "en", "es", "fa", "pt", "ar", "id")
DEFAULT_LANG = "en"

LANG_NATIVE_NAMES: dict[str, str] = {
    "ru": "Русский", "en": "English", "es": "Español", "fa": "فارسی",
    "pt": "Português", "ar": "العربية", "id": "Bahasa Indonesia",
}
# Английские имена — для языковой директивы LLM (см. summarizer).
LANG_ENGLISH_NAMES: dict[str, str] = {
    "ru": "Russian", "en": "English", "es": "Spanish", "fa": "Persian (Farsi)",
    "pt": "Portuguese", "ar": "Arabic", "id": "Indonesian",
}

_LOCALES_DIR = Path(__file__).resolve().parent / "locales"


def _load_locales() -> dict[str, dict[str, str]]:
    locales: dict[str, dict[str, str]] = {}
    for lang in SUPPORTED_LANGS:
        path = _LOCALES_DIR / f"{lang}.json"
        try:
            locales[lang] = json.loads(path.read_text(encoding="utf-8"))
        except Exception:  # noqa: BLE001
            logger.exception("i18n.locale_load_failed lang=%s path=%s", lang, path)
            locales[lang] = {}
    return locales


_LOCALES = _load_locales()


def normalize_language_code(code: str | None) -> str:
    """Telegram language_code (IETF, напр. 'pt-br') → поддерживаемый язык."""
    if not code:
        return DEFAULT_LANG
    primary = code.strip().lower().split("-")[0]
    return primary if primary in SUPPORTED_LANGS else DEFAULT_LANG


def t(key: str, lang: str, **fmt) -> str:
    """Перевод ключа: lang → en → сам ключ. Ошибка format не роняет вызов."""
    text = _LOCALES.get(lang, {}).get(key) or _LOCALES.get(DEFAULT_LANG, {}).get(key) or key
    if not fmt:
        return text
    try:
        return text.format(**fmt)
    except (KeyError, IndexError, ValueError) as exc:
        logger.warning("i18n.format_failed key=%s lang=%s error=%s", key, lang, exc)
        return text
```

- [ ] **Step 4: app/user_lang_store.py**

```python
"""Выбор языка пользователя (таблица user_langs).

source: 'auto' — детект из Telegram language_code при первом контакте;
'manual' — явный выбор через /language (auto его больше не перезаписывает).
"""
from __future__ import annotations

import time

from app.db import Database


class UserLangStore:
    def __init__(self, db: Database) -> None:
        self._db = db

    def get(self, user_id: int) -> tuple[str, str] | None:
        row = self._db.query_one(
            "SELECT lang, source FROM user_langs WHERE user_id = ?", (user_id,)
        )
        return (str(row["lang"]), str(row["source"])) if row else None

    def set(self, user_id: int, lang: str, source: str) -> None:
        self._db.execute(
            "INSERT OR REPLACE INTO user_langs(user_id, lang, source, updated_at) "
            "VALUES (?, ?, ?, ?)",
            (user_id, lang, source, time.time()),
        )
```
